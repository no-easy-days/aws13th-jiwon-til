## HTTPS Status
HTTP 상태 코드는 **첫 번째 숫자**에 따라 5가지 범주로 나뉩니다.

| 범주 | 이름 | 의미 | 비유 |
| --- | --- | --- | --- |
| **1xx** | Informational (정보) | 요청을 받았고, 처리 중입니다 | "네, 알겠습니다. 처리 중이에요" |
| **2xx** | Success (성공) | 요청이 성공적으로 처리되었습니다 | "완료됐습니다!" |
| **3xx** | Redirection (리다이렉션) | 요청 완료를 위해 추가 조치가 필요합니다 | "저쪽으로 가세요" |
| **4xx** | Client Error (클라이언트 오류) | 클라이언트의 요청에 문제가 있습니다 | "손님, 그건 안 돼요" |
| **5xx** | Server Error (서버 오류) | 서버에서 요청을 처리하지 못했습니다 | "죄송합니다, 저희 문제예요" |

## 자주 사용되는 상태 코드

실제로 자주사용하는 상태코드입니다.

### 2xx - 성공 응답

| 코드 | 이름 | 설명 | 사용 예시 |
| --- | --- | --- | --- |
| **200** | OK | 요청이 성공적으로 처리됨 | 웹 페이지 정상 로딩 |
| **201** | Created | 새로운 리소스가 생성됨 | 회원가입 완료, 게시글 작성 |
| **204** | No Content | 성공했지만 반환할 내용 없음 | 삭제 완료 |

### 3xx - 리다이렉션

| 코드 | 이름 | 설명 | 사용 예시 |
| --- | --- | --- | --- |
| **301** | Moved Permanently | 리소스가 영구적으로 이동됨 | 도메인 변경 |
| **302** | Found | 리소스가 임시로 이동됨 | 로그인 후 메인 페이지 이동 |
| **304** | Not Modified | 캐시된 버전 사용 가능 | 이미지 재요청 시 캐시 활용 |

### 4xx - 클라이언트 오류

| 코드 | 이름 | 설명 | 사용 예시 |
| --- | --- | --- | --- |
| **400** | Bad Request | 잘못된 요청 | 필수 입력값 누락 |
| **401** | Unauthorized | 인증 필요 | 로그인하지 않고 마이페이지 접근 |
| **403** | Forbidden | 권한 없음 | 관리자 페이지에 일반 유저 접근 |
| **404** | Not Found | 리소스를 찾을 수 없음 | 존재하지 않는 페이지 |
| **409** | Conflict | 충돌 발생 | 이미 존재하는 아이디로 가입 시도 |

### 5xx - 서버 오류

| 코드 | 이름 | 설명 | 사용 예시 |
| --- | --- | --- | --- |
| **500** | Internal Server Error | 서버 내부 오류 | 예상치 못한 서버 에러 |
| **502** | Bad Gateway | 게이트웨이 오류 | 프록시 서버 연결 실패 |
| **503** | Service Unavailable | 서비스 이용 불가 | 서버 점검 중 |

## 다형성
- 자바 언어의 다형성 활용
  - 역할 = 인터페이스
  - 구현 = 인터페이스를 구현한 클래스, 구현객체
  - 객체를 설계할 때에는 역할과 구현을 명확히 분리
