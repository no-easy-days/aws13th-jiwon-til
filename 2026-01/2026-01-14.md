## 동시성과 병렬성

### 동시성(Concurrency)

- con(together) + curr(run). “함께 달린다”
- 여러 작업이 함께 진행 중인 상태로, 일손은 하나지만 여러 작업을 번갈아 처리해서, 보는 입장에선 마치 동시에 돌아가는 것처럼 보입니다.

### 병렬성(Parallelism)

- para(beside) + allel(each other). “나란히 서로”
- 여러 사람이 각자 맡은 일을 진짜 동시에 처리합니다.(일손이 여러개라 가능)

---

### 동기(Synchronous)

- 앞 작업이 끝나야 다음 작업 시작 (시간을 맞춘다)

### 비동기(Asynchronous)

- 기다리는 동안 다른 일 가능 (시간을 안 맞춘다)

## 동시성(Concurrency) vs 병렬성(Parallelism)

두 개념 모두 "여러 일을 한꺼번에 처리한다"는 점에서 비슷해 보이지만 **용도가 완전히 다릅니다**.

<aside>

**한 줄 요약**

- **동시성**: 외부 서비스의 응답을 **기다리는** 동안 다른 일 하기
- **병렬성**: 큰 작업을 **조각내서** 여러 개를 동시에 처리하기
</aside>

## FastAPI

Python으로 API 서버를 빠르게 개발할 수 있도록 설계된 웹 프레임워크입니다.

### API(Application Programming Interface)

- 프로그램과 프로그램이 서로 대화하는 방식을 정의한 약속

## 코드 실행 방법

### 실행 단계

1단계: 코드를 파일에 저장

```python
from fastapi import FastAPI
app = FastAPI()

@app.get("/items/{item_id}")
async def read_item(item_id):
    return {"item_id": item_id}
```

2단계: 개발 서버 실행

```bash
fastapi dev [main.py](http://main.py)
```

## Path Parameter

- URL의 경로 안에 변수를 넣어서 동적으로 다른 리소스에 접근할 수 있게 만드는 것

## Query Parameter

- 서버에서 조건을 질문하는 매개변수, 요청의 범위나 조건을 설정

## 💭 오늘의 회고 (Retrospective)

- **Keep (좋았던 점)**: 몰입하는 시간이 늘어났습니다.
- **Problem (아쉬웠던 점)**: 이해하는 부분에 있어서 시간이 걸렸습니다.
- **Try (시도할 점)**: 빠르게 몰입해봐야겠습니다.
